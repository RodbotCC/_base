## Polished
Jake and Claude aren't just having a conversation — we're building a project with temporal continuity across three distinct memory layers:

1. **Claude's memory system** — persistent across chats, but summarized and lossy. Good for broad strokes and vibe.
2. **Project files** — durable within the Claude project, loaded into context when relevant.
3. **Jake's local filesystem** (`/Users/jakeaaron/the_infinity_problem`) — the ground truth. Timestamped by the OS, versioned, inspectable, portable, and lives outside Anthropic entirely.

The local filesystem is the anchor. If Claude's memory drifts or a summary goes fuzzy, we can cross-check against actual markdown files — real words, real dates, real sequence. That's temporal continuity with *evidence*, not just recollection. When we eventually red-team the Infinity Problem, we'll have a real paper trail of how the ideas evolved.

## Working conventions
- Insights get saved as individual markdown files in this directory.
- Filenames are descriptive slugs matching what the file is about — no dates (the OS tracks that).
- Each file has two sections: **Raw** (Jake's original stream-of-thought, preserved verbatim) and **Polished** (Claude's refined version).
- Neither version is canonical. The raw has texture, rhythm, and sideways associations the polish often smooths out. The polished has clarity. Both get preserved so we can mine either for the final work — which, fittingly, will never truly be final.
